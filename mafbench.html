MAFBench

A unified benchmark for evaluating multi-agent LLM frameworks

GitHub Repo Documentation

What MAFBench Measures

Orchestration

Measures overhead, latency, and throughput of agent scheduling and execution patterns.

Memory

Evaluates how frameworks store, retrieve, and share information between agents.

Planning

Tests reasoning accuracy and efficiency across different planning interfaces.

Specialization

Assesses how well frameworks enable agent specialization and role-based behavior.

Coordination

Evaluates multi-agent coordination success across different communication topologies.

Overhead

Measures framework-level overhead beyond direct LLM calls, including token costs and latency.

Why MAFBench Matters

Finally Compare Frameworks Fairly

Existing benchmarks focus on individual capabilities, but real-world performance depends on how frameworks integrate and orchestrate these capabilities.

MAFBench provides a standardized evaluation pipeline that isolates the impact of framework-level design choices, enabling meaningful comparisons across different architectural paradigms.

  • Unified execution pipeline across frameworks
  • Controlled evaluation isolating architectural effects
  • Comprehensive coverage of key dimensions

How It Works

1

Integrate Existing Benchmarks

MAFBench doesn't reinvent benchmarks. Instead, it integrates established, domain-relevant benchmarks (like AGENTSNET for coordination) within a unified execution pipeline.

2

Standardize Execution

All frameworks are evaluated using the same execution protocol, measurement methodology, and scoring criteria. This ensures fair, comparable results.

3

Isolate Framework Effects

By fixing the underlying LLM model, MAFBench attributes performance differences to framework architecture rather than model quality, revealing true architectural impact.

4

Generate Actionable Insights

Results translate directly into architectural design principles and framework-selection guidance, helping builders make informed decisions.
